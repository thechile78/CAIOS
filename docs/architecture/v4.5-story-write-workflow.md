# CAIOS v4.5 Validated Story Write Workflow

This milestone adds authenticated story creation through a database RPC that performs the story insert and audit-event insert in one transaction.

## Safety model

- Only active administrator, editor, producer, or researcher roles may create stories.
- The database derives the actor from `auth.uid()`; the browser cannot choose `created_by` or `updated_by`.
- New stories always begin in `discovered` status.
- Title, desk, priority, summary, and body are validated before the RPC call and again in PostgreSQL.
- The RPC records a `story.created` audit event in the same transaction.
- No approval, WordPress draft, publishing, scheduling, or service-role access is added.

## Staging verification

Apply the migration only to a non-production Supabase project. Test one account for every role, confirm read-only and reviewer accounts cannot create stories, confirm invalid fields fail, and confirm each successful story has exactly one matching audit event.
