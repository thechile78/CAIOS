# CAIOS v4.8 Approval Queue and Internal Handoff

## Scope

This milestone provides a single authenticated queue for stories awaiting human review and stories already approved. Approved stories may be placed into an internal packaging handoff. The handoff does not contact WordPress, create a publication record, schedule content, or publish anything.

## Security invariants

- Supabase RLS remains enabled.
- Actor identity is derived from `auth.uid()`.
- Only administrator, editor, and producer roles may request a handoff.
- A handoff requires an `approved` story, complete checklist, human approval, and a matching immutable approval record.
- Optimistic concurrency rejects stale requests.
- A partial unique index prevents duplicate active handoffs.
- The handoff row and audit event are written atomically.

## Staging verification

1. Apply migrations through v4.8 to a non-production Supabase project.
2. Create test users for every role.
3. Confirm awaiting-approval stories appear in the queue.
4. Confirm only reviewers can make approval decisions through the existing review page.
5. Confirm approved stories appear with handoff controls only for authorized roles.
6. Confirm incomplete, stale, non-approved, and duplicate handoff requests fail.
7. Confirm no request reaches WordPress or any external publishing system.

## Rollback

Before production rollout, create and test a rollback migration that revokes the RPC, drops the partial index and RLS policies, drops `editorial_handoffs`, and then drops `handoff_state`. Preserve audit records through a backup before rollback.
