# CAIOS v5.0 — WordPress draft outbox foundation

## Purpose

Prepare a reviewed internal WordPress draft intent from an immutable CAIOS editorial package. This milestone does not contact WordPress.

## Safety guarantees

- The feature is disabled unless `CAIOS_WORDPRESS_DRAFT_OUTBOX_ENABLED=true` is explicitly set.
- Only administrators and editors may create an outbox record.
- The source must be an immutable editorial package whose story snapshot is approved and whose checklist snapshot records human approval.
- The payload status is fixed to `draft`.
- Scheduling fields, author overrides, and credential-like fields are rejected.
- One queued outbox intent is allowed per package.
- Package freshness is checked using the immutable package creation timestamp.
- The outbox record and audit event are created in one database transaction.
- No authenticated update or delete policy is added.
- No HTTP client, WordPress URL, username, application password, token, or service-role browser access is introduced.

## Staging verification

1. Apply migrations through v5.0 to a non-production Supabase project.
2. Keep `CAIOS_WORDPRESS_DRAFT_OUTBOX_ENABLED` unset and verify the queue action is blocked.
3. Confirm producer, researcher, reviewer, and read-only roles cannot queue an intent.
4. Confirm a stale package timestamp is rejected.
5. Confirm a package without approved snapshots is rejected.
6. Set the feature flag to `true` only in staging.
7. Queue one internal draft intent and inspect the payload and audit event.
8. Confirm a duplicate queued intent is rejected.
9. Confirm no external network request occurs.
10. Return the feature flag to disabled after validation.

## Explicit exclusions

This milestone does not add WordPress credentials, a WordPress endpoint, draft delivery, media upload, scheduling, publishing, deployment, production secrets, or automatic execution.
