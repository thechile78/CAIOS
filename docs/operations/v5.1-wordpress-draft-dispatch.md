# CAIOS v5.1 — Manual WordPress draft dispatch

## Safety model

The adapter is server-only, manually triggered, disabled by default, and dry-run by default. It accepts only outbox payloads whose WordPress status is `draft`. Scheduling, author override, password fields, automatic retries, media uploads, and publishing are excluded.

## Required staging variables

- `CAIOS_WORDPRESS_DRAFT_DISPATCH_ENABLED=true`
- `CAIOS_WORDPRESS_DRAFT_DRY_RUN=true` for initial verification
- `CAIOS_WORDPRESS_SITE_URL=https://staging.example.com`
- `CAIOS_WORDPRESS_USERNAME`
- `CAIOS_WORDPRESS_APPLICATION_PASSWORD`

Credentials must be stored only in the server-side secret manager. Never expose them through `NEXT_PUBLIC_` variables.

## Staging sequence

1. Apply migrations through v5.1 to a non-production Supabase project.
2. Keep dry-run enabled and verify administrator/editor access only.
3. Confirm queued and failed records can be claimed once using a fresh `updated_at` value.
4. Confirm processing, sent, and cancelled records cannot be claimed.
5. Confirm audit events are written for start, success, and failure.
6. Configure a dedicated staging WordPress account with draft-creation permissions only.
7. Disable dry-run in staging and create one test draft.
8. Confirm the WordPress post remains `draft` and is not scheduled or published.
9. Rotate the staging application password after acceptance testing.

## Explicit exclusions

This milestone does not enable production dispatch, background workers, automatic retries, publishing, scheduling, media uploads, deployment, or production credentials.
