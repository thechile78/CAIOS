# CAIOS v4.9 — WordPress draft-package preparation

This milestone prepares immutable, internal content snapshots for later reviewed transfer into WordPress.

## Safety boundary

- No WordPress URL, username, application password, token, or API client is configured.
- No network request is made to WordPress.
- No post is created, scheduled, updated, or published.
- The exported JSON always declares `status: draft`.
- A human must review and manually transfer the package.

## Eligibility

A package can be prepared only when the story is approved, all editorial checks are complete, human approval is recorded, the immutable approval record matches, the internal handoff is active, the caller has an authorized role, and the optimistic concurrency timestamp is current.

## Verification

Apply the migration first in a non-production Supabase project. Test successful packaging, stale-write rejection, duplicate prevention, unauthorized roles, incomplete checklists, and missing approval records before considering any future WordPress connector.
